# K230 坐姿识别 - 当日复盘（2026-02-22）

## 1) 今日目标
- 提升侧面演示场景下的识别稳定性与可演示性。
- 重点处理问题：
  - `head_down / hunchback / tilt` 混淆。
  - 状态粘连与“切换后不更新”的主观表现。
  - 最终展示口径统一（简化类别）。

## 2) 今日完成内容
- 完成代码排查与日志对齐：
  - 确认 `no_person` 触发路径来自人体门控（置信度/框面积/有效关键点）。
  - 确认异常状态由风险平滑与门控阈值共同决定。
- 多轮姿态特征与门控逻辑调参/改造（含回滚）。
- 增加并输出调试字段：
  - `posture_debug` 扩展（分数、delta、进入/退出原因等）。
  - 保留细粒度识别信息用于定位。
- 最终按演示需求完成输出口径改造：
  - 对外三类：`normal / bad / no_person`。
  - 对内保留细类：`posture_type_fine`（如 `head_down/hunchback/tilt`）。

## 3) 遇到的问题与现象
- 侧面机位下，`head_down` 与 `hunchback` 在 2D 投影中重叠明显，细分类可分性不足。
- 多轮阈值调参容易出现两极：
  - 过敏：大量误报。
  - 过钝：几乎全是 `normal`。
- 使用静态图片切换时，因缺少自然过渡与深度信息，边界动作更容易误判或不触发。
- 基线/状态机制在特定顺序输入下可能被“污染”，导致后续判断偏保守。

## 4) 根因判断（复盘结论）
- 主要不是图片清晰度问题，而是**机位几何约束 + 2D关键点表达能力上限**。
- 侧面演示中，“轻微低头/轻微驼背/轻微侧倾”的边界本就难分，且对阈值非常敏感。
- 单一风险分数驱动时，类别抢占和状态滞后更明显，演示体验波动大。

## 5) 已采取的收敛策略
- 放弃继续在同一逻辑上盲目加阈值，改为“可回退、可解释”的策略：
  - 保留 legacy 稳定路径。
  - 输出层做演示友好映射（3类）降低展示抖动。
  - 内部细类与调试字段保留，确保后续可持续优化。

## 6) 当前版本状态（交付口径）
- 演示输出：
  - `posture_type`: `normal / bad / no_person`
- 调试输出：
  - `posture_type_fine`: 细分类（`head_down/hunchback/tilt/...`）
  - `posture_debug`: 详细分数与状态字段
- 目标达成：
  - 演示稳定性优先，减少“细分类抖动”对展示的影响。

## 7) 建议的演示规范（侧面固定机位）
- 机位高度对齐肩-胸中线，避免明显俯拍/仰拍。
- 人体在画面占比保持稳定（头到髋完整可见）。
- 侧面夹角尽量固定在 80°~90°。
- 先正常 2~3 秒，再切异常姿势，减少基线污染。
- 选用“明显动作”而非轻微动作进行演示（强低头、强驼背）。

## 8) 后续迭代建议
- 演示阶段：继续沿用三类对外口径。
- 研发阶段：用 `posture_type_fine + posture_debug` 做离线样本复盘，分场景建阈值档位（正侧面/半侧面）。
- 若后续允许更大改动：考虑引入轻量时序模型或多视角方案提高细分类上限。
