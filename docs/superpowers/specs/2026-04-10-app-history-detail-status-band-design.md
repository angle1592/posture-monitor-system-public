# App 当天明细状态时间带设计

## 1. 背景

当前历史页已经把：

- `7 天趋势` 做成了曲线
- `当天明细` 改成了 canvas 渲染

但当天明细仍然沿用“0/100 折线”表达离散姿态状态，导致在实机上呈现为明显的“梳子图”：

1. 大量竖向跳变
2. 密集节点堆积
3. `5/10/15 分钟` 聚合差异不自然

这不是样式微调能解决的问题，而是图形类型与数据语义不匹配。

## 2. 目标

1. 保留 `7 天趋势` 的曲线表达。
2. 将 `当天明细` 改为更适合离散状态的“状态时间带”。
3. 继续保留：
   - 自适应时间轴
   - `no_person` 不计入评分
   - `5/10/15 分钟` 聚合切换

## 3. 设计原则

### 3.1 图形职责分离

- `7 天趋势` 用曲线：表达整体趋势
- `当天明细` 用时间带：表达具体时段状态

### 3.2 状态语义优先

当天明细不是连续数值曲线，而是离散类别序列：

- `normal`
- `abnormal`
- `no_person`

因此更适合用横向时间带展示，而不是上下跳动的折线。

## 4. 状态时间带方案

### 4.1 基础形态

当天明细区域改成一条横向时间带：

- 绿色段：正常
- 橙色段：异常
- 透明断开：无人

### 4.2 聚合桶到状态段

每个时间桶根据聚合结果转为一个状态：

1. `score === null`
   - 视为 `gap`
   - 不绘制填充段

2. `normalCount > 0 && abnormalCount === 0`
   - 视为 `normal`

3. `abnormalCount > 0 && normalCount === 0`
   - 视为 `abnormal`

4. `normalCount > 0 && abnormalCount > 0`
   - 视为 `mixed`
   - 为了避免信息丢失，绘制为双色渐变段

### 4.3 连续段合并

相邻且状态一致的桶应合并成一个连续时间段，以减少碎片化渲染。

这能显著改善视觉效果，并让 `10/15 分钟` 聚合下的差异更自然。

## 5. 时间轴

当天明细继续使用自适应时间轴：

- 起点：最早有效数据附近
- 中点：时间窗中点
- 终点：最晚有效数据附近

底部标签动态显示该时间范围，而不是固定 `00:00 / 12:00 / 24:00`。

## 6. 影响范围

主要修改：

- `app/src/utils/historyChart.ts`
- `app/src/utils/historyChart.test.ts`
- `app/src/pages/history/index.vue`

## 7. 非目标

1. 不改 7 天趋势曲线。
2. 不改评分口径。
3. 不引入第三方图表库。
